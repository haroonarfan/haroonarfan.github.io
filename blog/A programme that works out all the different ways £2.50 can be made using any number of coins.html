A programme that works out all the different ways £2.50 can be made using any number of coins

by Haroon Arfan  16/06/2020
3 min read

It is possible to make £2.50 in the following way:

  • 1×£1 + 2×50p + 2×20p + 1×5p + 1×2p + 3×1p

"""
If used for loop this would have crashed the program or raised printing error.
So used recursive approach where we call the function without having the solution calculated before stored in memory.
Used yield instead of return to save memory.
"""


def change(money, coins_available, coins_so_far=[]):
    if sum(coins_so_far) == money:
        yield coins_so_far  # Used yield as the list would have taken so much memory and probably raised a memory
    # error.
    elif sum(coins_so_far) > money:  # If exceeded the amount
        pass
    elif not coins_available:  # If no coins are given
        pass
    else:
        for c in change(money, coins_available[:], coins_so_far + [coins_available[0]]):  # Using recursive approach
            yield c  # Used yield as the list would have taken so much memory and probably raised a memory error.
        for c in change(money, coins_available[1:], coins_so_far):  # Using recursive approach
            yield c  # Used yield as the list would have taken so much memory and probably raised a memory error.


n = 250
coins = [1, 2, 5, 20, 50, 100]

count = 0

solutions = [s for s in change(n, coins)]  # To convert generator into a list using list-comprehension
for s in solutions:  # To print individual list instead of nested list
    count += 1
print(f"There are {count} ways to make change for £{n} using", *coins, ".")

print('optimal solution:', min(solutions, key=len))  # To print optimal solution
